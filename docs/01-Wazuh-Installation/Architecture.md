# Architecture

## Objective

The purpose of this architecture is to establish a centralized Security Information and Event Management (SIEM) platform capable of collecting and analyzing security events generated by Windows endpoints.

The Wazuh Manager serves as the central component responsible for receiving endpoint telemetry, processing security events, generating alerts, and providing visibility through the Wazuh Dashboard.

---

# High-Level Architecture

```
+-------------------+
| Windows Endpoint  |
|   Wazuh Agent     |
+---------+---------+
          |
          | Security Events
          |
+---------v---------+
|   Wazuh Manager   |
+---------+---------+
          |
          | Alert Processing
          |
+---------v---------+
| Wazuh Dashboard   |
+---------+---------+
          |
          |
+---------v---------+
| Security Analyst  |
+-------------------+
```

---

# Components

## Windows Endpoint

The Windows endpoint generates operating system and security-related events.

The Wazuh Agent collects relevant logs and securely forwards them to the Wazuh Manager.

---

## Wazuh Agent

The Wazuh Agent is responsible for:

- Collecting endpoint logs
- Monitoring system activity
- Forwarding telemetry
- Maintaining communication with the manager

---

## Wazuh Manager

The Wazuh Manager receives endpoint telemetry and performs centralized processing.

Primary responsibilities include:

- Event processing
- Rule evaluation
- Alert generation
- Agent management

---

## Wazuh Dashboard

The dashboard provides centralized visibility into collected security events.

Analysts can:

- View alerts
- Investigate events
- Monitor endpoint status
- Search logs

---

## Security Analyst

The Security Analyst reviews alerts, investigates suspicious activity, validates security events, and performs incident response based on the collected telemetry.

---

# Security Flow

1. Windows generates a security event.
2. The Wazuh Agent collects the event.
3. The event is forwarded to the Wazuh Manager.
4. The Manager processes the event.
5. Alerts become visible within the Wazuh Dashboard.
6. The Security Analyst investigates the event.

---

# Enterprise Value

This architecture provides:

- Centralized security monitoring
- Endpoint visibility
- Log collection
- Alert generation
- Incident investigation
- Foundation for future detection engineering
